# はじめに

本ドキュメントはリポジトリに公開されているHTML (`index.html`) とCSS (`style.css`) を基に、このコードを別の団体やサイトに移植・活用するための手順を解説しています。

画像、文章などは事例として仮設定していますので、実際には適切なものと差し替えてください。

また連絡先等のリンクについても同様に適切なURL等に差し替えをお願いします。

-----

## 1\. サイトの概要と特徴

このコードは、大学のスポーツチームが新入生を勧誘するために設計された、**シングルページ**形式のウェブサイトです。

| 項目 | 詳細 |
| :--- | :--- |
| **サイト形式** | シングルページ (One-page structure) |
| **主なテーマ** | 「EAGLES」（赤と黒を基調とした力強さ） |
| **特徴** | ヒーローセクション、チームの紹介、提供価値、先輩メッセージ、詳細なイベントスケジュール、モバイル対応（ハンバーガーメニュー）。 |
| **技術要素** | HTML5, CSS3, JavaScript (簡単なモバイルメニュー制御) |

-----

## 2\. 必要なファイル構成

コードをサーバーにアップロードする際、以下のファイルとフォルダ構造を**維持**する必要があります。

```
/ (ルートディレクトリ)
├── index.html           <- サイトの本体
├── statics/
│   ├── css/
│   │   └── style.css    <- スタイルシート
│   └── img/             <- 画像ファイルを格納するフォルダ
│       ├── team_logo_eagles.webp 
│       ├── qb.webp 
│       ├── dl.webp 
│       ├── as.webp 
│       ├── mg.webp 
│       └── top_img.webp (ヒーローセクション背景画像)
└── [その他、別途必要な画像ファイル]
```

## 3\. HTML (`index.html`) のカスタマイズ

`index.html` はサイトの**コンテンツと構造**を定義しています。チームの情報に合わせて内容を書き換えてください。

### 3.1. `<head>`と外部ファイル

| 項目 | 目的と変更点 |
| :--- | :--- |
| **タイトル** | `<title>EAGLES - 広島鯉城大学アメリカンフットボール部</title>` を新しいチーム名に変更します。 |
| **ファビコン** | `<link rel="icon" href="statics/img/team_logo_eagles.webp">` の画像ファイルを新しいロゴに差し替えてください。 |
| **CSS** | `<link rel="stylesheet" href="statics/css/style.css">` のパスはそのまま維持してください。 |
| **アイコン** | Font Awesome (CDN) を使用しています。インターネット接続があればSNSアイコンなどは動作します。 |

### 3.2. 主要なセクションの変更点

サイトは、アンカーリンク (`#about`, `#schedule`など) に対応したセクションに分かれています。

| セクションID | 内容 | カスタマイズのポイント |
| :--- | :--- | :--- |
| なし | **ヒーローセクション** | `<p class="topline">`、`<h1>`（チーム名）、`<p class="tagline">`（キャッチコピー）のテキストを書き換えます。 |
| `#about` | **チーム紹介** | テキスト内容を新しいチームの歴史やモットーに書き換えます。 |
| `#team` | **提供価値（特長）** | `.feature-card` 内の `<h3>` と `<p>` を新しいチームの特長に書き換えます。 |
| `#message` | **先輩部員メッセージ** | `alumni-card` 内の氏名、ポジション、写真ファイル (`src`)、引用、メッセージをすべて新しい部員に差し替えます。**画像ファイルは `statics/img/` フォルダに配置してください。** |
| `#schedule` | **新歓スケジュール** | `.schedule-card` 内の日付、タイトル、タグ、詳細をすべて書き換えます。 |
| `#join` | **参加を促すCTA** | テキストの書き換えと、`btn-primary` のリンク先を確認・設定します。 |
| `#sns` | **フッター（SNSリンク）** | `social-links` 内の `href="#"` を公式SNSのURLに書き換えます。 |

### 3.3. JavaScript（モバイルメニュー）

以下のJavaScriptは、スマートフォン表示時にハンバーガーメニューを動作させるために使われています。

  * **役割:** ハンバーガーアイコンをクリックするとメニューが表示され、同時に背景のスクロールを止める機能です。
  * **移植時の注意点:** HTMLのID (`menu-toggle`, `menu-list`) やクラス名 (`is-active`, `is-open`) を変更しなければ、**このコードはそのまま利用できます**。

```javascript
// index.htmlの最後にあるJavaScriptコード
document.addEventListener('DOMContentLoaded', () => {
    const menuToggle = document.getElementById('menu-toggle');
    const menuList = document.getElementById('menu-list');
    const body = document.body;

    menuToggle.addEventListener('click', () => {
        // ... アニメーションとメニューの表示/非表示を切り替える処理 ...
    });

    // メニュー項目をクリックしたときにメニューを閉じる処理
    menuList.querySelectorAll('a').forEach(link => {
        // ...
    });
});
```

## 4\. CSS (`style.css`) のカスタマイズ

`style.css` はデザインの「見た目」を制御しています。主に**配色**と**背景画像**を調整します。

### 4.1. 配色のテーマ変更

このサイトは主に「**赤色** (`#ff4757`)」と「**黒・濃いグレー** (`#0a0a0a`, `#1a1a1a`)」で構成されています。

| スタイル | 色の役割 |
| :--- | :--- |
| `#ff4757` | メインのアクセントカラー（赤）。ロゴ、CTAボタン、ボーダー、スケジュールカードなどで使用。 |
| `#0a0a0a` | 背景の濃い黒色。 |
| グラデーション | `.hero h1` や `.logo` など、テキストにグラデーションを適用しています。 |

**移植・カスタマイズのヒント:**
チームカラーを変更したい場合は、`style.css`内の\*\*`#ff4757`\*\*を、新しいカラーコードに一括で置き換えることで、サイト全体のアクセントカラーが変更されます。

### 4.2. ヒーローセクションの背景画像

ヒーローセクション (`.hero`) の背景画像は、CSSの以下の部分で指定されています。

```css
.hero {
    /* ... 略 ... */
    background-image: url('/statics/img/top_img.webp'); /* 👈 ここを新しい画像パスに変更 */
    background-size: cover;          
    background-position: center;     
    /* ... 略 ... */
}
```

  * 新しい背景画像を使用する場合は、`url('')` 内の**ファイルパスを正しいものに書き換えてください**。
  * `.hero::before` で設定されている **黒色の半透明なオーバーレイ** (`opacity: 0.6`) を調整することで、背景画像の明るさや文字の読みやすさを変更できます。

### 4.3. スケジュールカードのカラーコード

スケジュールカードのタグには、重要度を示すためのカスタムクラスが設定されています。

| クラス名 | 意味 |
| :--- | :--- |
| `.schedule-tag.is-warning` | 警告・必須事項 (`#ff9800`) |
| `.schedule-tag.is-success` | 成功・交流イベント (`#2ecc71`) |
| `.schedule-tag.is-danger` | 危険・重要事項 (`#e74c3c`) |
| `.schedule-tag.is-accent` | メインカラーで強調 (`#ff4757`) |

### 4.4. レスポンシブデザイン

`@media (max-width: 768px)` のブロック内で、スマートフォン表示時のレイアウト（モバイルフレンドリーなレイアウト）が定義されています。

  * スケジュール (`.schedule-grid`) などが横並びから縦積みに変更されています。
  * モバイル対応のデザインを大きく変更する必要がなければ、この部分はそのまま維持してください。

### 5.利用に際して

本コード群は新入生に向けての情報発信用の臨時ランディングページという位置づけで、逐次情報を提供することは想定しておりません。そのため細かい情報発信は別途部の公式SNSまたは公式サイトでの対応をご検討ください。

本コード群をGithubで管理することによりGithub pagesやCloudflare Pagesなどのサービスからサイトを無料で公開することができます。ただしそのようなサービスを利用する場合には接続先URL名の設定には制限があり大学の公式ドメイン以外で動作するため、実際の公開に際しては学内の関連部門へ相談され適切な環境での公開をされることを強く推奨いたします。

リポジトリのコピー方法は**付録:GithubリポジトリをZIP形式でダウンロードする手順**を参照してください。

## 付録:GithubリポジトリをZIP形式でダウンロードする手順

### 1. GitHubリポジトリページに移動

まず、ダウンロードしたいリポジトリのGitHubページをブラウザで開きます。

### 2. 「<> Code」ボタンをクリック

リポジトリのファイルリストが表示されているエリアの右上に、緑色（またはテーマに応じた色）の**「<> Code」**と書かれたボタンがあります。これをクリックしてください。

### 3. 「Download ZIP」を選択

ドロップダウンメニューが開きます。

* このメニューには通常、「HTTPS」「SSH」といったクローン用のURLと並んで、一番下に**「Download ZIP」**というオプションが表示されます。
* これをクリックすると、リポジトリ内の**全ファイル（`.git`ディレクトリを除く）**が`.zip`ファイルとして自動的にダウンロードされます。

これで、リポジトリの最新の状態がローカルコンピューターに`zip`ファイルとしてコピーされます。
